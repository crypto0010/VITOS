# VITOS Wallpaper + Curated App Menu — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the residual Kali desktop wallpaper with a VITOS one everywhere, and add native "VITOS · Security" and "VITOS · AI" submenus to the XFCE menu listing every (sandboxed) security tool plus the Ollama/Gemma chat and the Dashboard.

**Architecture:** Per-tool `.desktop` launchers (Categories `X-VITOS-Security`/`X-VITOS-AI`) + two `.directory` files + an `applications-merged/vitos.menu` merge shipped in the `vitos-tools` package; launchers route through the existing `/usr/local/bin/<tool>` → `vitos-run` sandbox/telemetry wrapper via a small `vitos-launch-tool` helper. Wallpaper PNG is generated by the existing `build-branding.sh` and applied via an `/etc/skel` XFCE backdrop config plus a login-time `xfconf-query` safety net, both shipped through `includes.chroot` so they override Kali with no dpkg file-conflict. A new `verify-appmenu` CI job validates everything in a Kali container; no ISO build.

**Tech Stack:** POSIX sh, freedesktop `.desktop`/`.directory`/XDG `.menu`, XFCE `xfconf`, ImageMagick (`convert`), Debian packaging (`debhelper`), GitHub Actions (Kali container).

**Branch:** `feat/vitos-wallpaper-app-menu` (already created; spec committed there).

**Reference spec:** `docs/superpowers/specs/2026-06-09-vitos-wallpaper-and-app-menu-design.md`

### Task 1: `vitos-launch-tool` helper

**Files:**
- Create: `vitos-v1/packages/vitos-tools/usr/bin/vitos-launch-tool`

- [ ] **Step 1: Write the helper**

Create `vitos-v1/packages/vitos-tools/usr/bin/vitos-launch-tool` with exactly:

```sh
#!/bin/sh
# VITOS application launcher helper.
# Shipped in vitos-tools; invoked by the VITOS · Security / VITOS · AI .desktop
# launchers (Terminal=true). Keeps the .desktop files trivial and centralizes
# how security tools open in a terminal.
#
# Usage: vitos-launch-tool <tool> [--repl]
#   <tool>   a name from vitos-tools' wrapped-tool list (resolves to the
#            sandboxed /usr/local/bin/<tool> -> vitos-run), OR the special
#            name 'ollama' (the local Gemma 3 chat).
#   --repl   the tool is interactive (msfconsole, r2, scapy, ...): exec it
#            directly. Without --repl the tool is a one-shot CLI: print its
#            --help, then drop into an interactive shell with the wrapped tool
#            on PATH so the student can run it with arguments.
#
# Set VITOS_LAUNCH_TEST=1 to print the resolved plan and exit 0 (used by
# tests/verify-appmenu.sh — never execs anything).
set -u

TOOL="${1:-}"
[ -n "$TOOL" ] || { echo "usage: vitos-launch-tool <tool> [--repl]" >&2; exit 2; }
shift
MODE="cli"
[ "${1:-}" = "--repl" ] && MODE="repl"

case "$TOOL" in
    ollama) RUN="ollama run vitos-intent"; MODE="repl" ;;
    *)      RUN="/usr/local/bin/$TOOL" ;;
esac

if [ "${VITOS_LAUNCH_TEST:-0}" = "1" ]; then
    echo "TOOL=$TOOL MODE=$MODE RUN=$RUN"
    exit 0
fi

printf '\033[1;36m== VITOS · %s ==\033[0m\n' "$TOOL"
echo "Sandboxed via vitos-run (session telemetry on). Press Ctrl-D or type 'exit' to close."
echo

if [ "$MODE" = "repl" ]; then
    # Interactive tool: run it, then leave a shell so the window doesn't vanish.
    $RUN
    echo
    echo "[$TOOL exited — shell ready]"
else
    # One-shot CLI: show help, then an interactive shell (wrapped tool on PATH).
    $RUN --help 2>/dev/null || $RUN -h 2>/dev/null || true
    echo
fi
exec "${SHELL:-/bin/bash}" -i
```

- [ ] **Step 4: Create the login-time enforcement script**

```bash
cd /c/Users/HP/Documents/VITOS
BIN=vitos-v1/live-build/config/includes.chroot/usr/local/bin
mkdir -p "$BIN"
cat > "$BIN/vitos-set-wallpaper" <<'EOF'
#!/bin/sh
# VITOS wallpaper enforcement — run from /etc/xdg/autostart on XFCE login.
# Forces the VITOS wallpaper on every backdrop property xfconf knows about
# (overrides any stale Kali value left in a per-user xfconf db). Idempotent.
WP=/usr/share/vitos/branding/wallpaper.png
[ -f "$WP" ] || exit 0
command -v xfconf-query >/dev/null 2>&1 || exit 0

# Update every existing backdrop image property to the VITOS wallpaper.
xfconf-query -c xfce4-desktop -l 2>/dev/null \
  | grep -E '/backdrop/.*/(last-image|image-path)$' \
  | while read -r prop; do
        xfconf-query -c xfce4-desktop -p "$prop" -s "$WP" 2>/dev/null || true
    done

# Ensure at least the primary monitor is set even on a fresh profile.
xfconf-query -c xfce4-desktop \
  -p /backdrop/screen0/monitor0/workspace0/last-image -s "$WP" 2>/dev/null \
  || xfconf-query -c xfce4-desktop \
       -p /backdrop/screen0/monitor0/workspace0/last-image -n -t string -s "$WP" 2>/dev/null \
  || true

# Zoomed style (5) wherever an image-style property exists.
xfconf-query -c xfce4-desktop -l 2>/dev/null \
  | grep -E '/backdrop/.*/image-style$' \
  | while read -r prop; do
        xfconf-query -c xfce4-desktop -p "$prop" -s 5 2>/dev/null || true
    done

exit 0
EOF
chmod +x "$BIN/vitos-set-wallpaper"
```

- [ ] **Step 1: Add the volatility3 install + shim block**

In `vitos-v1/live-build/config/hooks/normal/9000-firstboot.hook.chroot`, insert this block immediately before the `passwd -l root || true` line:

```sh
# volatility3 — not in Kali main; install via pip so the forensics launcher
# works fully offline on lab machines (baked into the squashfs at build time).
# Best-effort: never fail the build (mirrors the firejail-vendoring philosophy).
# The pip console script is `vol`, but vitos-run dispatches on the name
# 'volatility3' (the symlink basename), so we also drop a /usr/bin/volatility3
# shim that execs `vol`.
if ! command -v vol >/dev/null 2>&1; then
    pip3 install --break-system-packages volatility3 >/dev/null 2>&1 \
      || pipx install volatility3 >/dev/null 2>&1 \
      || echo "WARNING: volatility3 install failed (forensics launcher will report 'not installed')"
```
