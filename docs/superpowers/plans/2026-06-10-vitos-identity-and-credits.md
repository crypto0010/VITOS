# VITOS Identity & Credits (PR A) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebrand every user-facing OS version/identity string to VITOS + VIT Bhopal (no "Kali"), and surface the team, a Special Thanks, and the punchline through a GUI "About VITOS" page and a terminal `vitos-credits` command.

**Architecture:** VITOS identity files (`os-release`, `lsb-release`, `issue`, `motd`) ship via `includes.chroot` (copied after packages → override the Kali base, no dpkg conflict). The dashboard `About.tsx` is rebranded in place. Credits live in the `vitos-tools` package as an offline self-contained `vitos-about.html` (opened via `xdg-open`) and a POSIX `vitos-credits` script, both under a new **VITOS · About** menu category. A Kali-container CI job (`verify-branding`) validates identity, punchline, and a credits drift-guard parsed from `website/index.html`.

**Tech Stack:** POSIX sh, freedesktop `.desktop`/`.directory`/XDG `.menu`, static HTML/CSS (offline, system fonts), Debian packaging, GitHub Actions (Kali container).

**Branch:** `feat/vitos-identity-credits` (already created off `feat/vitos-wallpaper-app-menu`; spec committed there). PR A base = `feat/vitos-wallpaper-app-menu`.

**Reference spec:** `docs/superpowers/specs/2026-06-10-vitos-branding-identity-ai-design.md` (Component A).

**Scope note:** The punchline on the **wallpaper / greeter / Plymouth / lock** images is generated by `build-branding.sh`, which is overhauled in **PR B** — so those punchline surfaces land in Plan B, not here. PR A covers the punchline on the **textual** surfaces (`issue`, `motd`, About page, credits) only.

**TDD adaptation:** Static-asset/packaging feature with no runnable unit on the dev host (Windows, no Linux/ISO build). Each task carries the strongest *locally runnable* check (POSIX `sh -n`, `grep`, os-release parse). The authoritative integration gate is `verify-branding.sh` (Task 7), proven green in the Kali CI container (Task 9) — mirroring `verify-appmenu.sh`.

### Task 4: Terminal credits — `vitos-credits`

**Files:**
- Create: `vitos-v1/packages/vitos-tools/usr/bin/vitos-credits`

- [ ] **Step 1: Write the script**

Create the file with EXACTLY this content:

```sh
#!/bin/sh
# VITOS terminal credits — prints the team, Special Thanks, and the punchline.
# Shipped in vitos-tools; launched by vitos-credits.desktop (Terminal=true) and
# runnable directly. Pure POSIX sh + ANSI color (degrades to plain text).
set -u

C=''; M=''; D=''; R=''
if [ -t 1 ]; then C='\033[1;36m'; M='\033[1;35m'; D='\033[2m'; R='\033[0m'; fi

printf "%b" "$C"
cat <<'BANNER'
 __     __ ___ _____  ___   ____
 \ \   / /|_ _|_   _|/ _ \ / ___|
  \ \ / /  | |  | |  | | | |\___ \
   \ V /   | |  | |  | |_| | ___) |
    \_/   |___| |_|   \___/ |____/
BANNER
printf "%b" "$R"

printf "  ${C}VITOS v1.0.2${R} — VIT Operating System for Cybersecurity\n"
printf "  ${M}Designed at VIT Bhopal for VITians${R}\n"
printf "  ${D}Designed and developed at VIT Bhopal — Cybersecurity and Digital Forensics Lab${R}\n\n"

printf "  ${C}== Project Leadership ==${R}\n"
printf "  Project Director ............ Dr. Hemraj Shobharam Lamkuche\n"
printf "  Chief Mentor ................ Dr. Pon Harshavardhanan\n"
printf "  Division Head (CDF) ......... Dr. Saravanan D.\n\n"

printf "  ${C}== Contributing Team ==${R}\n"
for n in \
  "Matrupriya Dibyanshu Panda" "Spandan Gope" "Bharat Raghuvanshi" \
  "Mayank Singh Bhadouria" "Advait Sahu" "Aayushman Arora" "Harsh Singh" \
  "Satyanarayana Murthy V" "Ravi Shankar" "Agnibha" "Leonardo" "Mannat Pal" \
  "Ambika" "Rashmi" "Nahal" "Piyush" "Krishno"; do
    printf "   • %s\n" "$n"
done
printf "\n"

printf "  ${C}== Special Thanks ==${R}\n"
printf "  Hon'ble Ms. Kadhambari S. Viswanathan — for her support.\n\n"

printf "  ${D}© 2026 VIT Bhopal University${R}\n"
```
